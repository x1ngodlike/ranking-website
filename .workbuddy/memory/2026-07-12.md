## 已实施 P0 优化（用户确认改动不影响线上后执行）
1. **原子写**：`server/server.js` 的 `writeJsonFile` 改为「先写 `.tmp` 再 `renameSync`」，杜绝中途崩溃导致的半截 JSON 数据损坏。
2. **写入串行化**：新增 `writeChain` Promise 链 + `serializedWrite`，所有 data/matches/auth/backup 写入排队执行，避免并发请求交错写同一文件。同步改了相关路由为 `async/await`（`/api/data`、`/api/matches`、`/api/admin/password`、`/api/admin/clear-data`、`/api/admin/backups/create`、`/api/admin/backups/restore` 及 `writeData/writeMatches/writeAuth/createBackup/restoreBackup/runMatchSync` 等）。
3. **前端保存防抖**：`src/store/useAppStore.ts` 的 `saveToServer` 改为 600ms 尾部防抖（始终以最新 state 落盘），新增 `beforeunload` 时刷盘；调用点 `saveToServer(get())` 无需改动。
